Summary

This section marks our first real dive into Java programming. We take the traditional "Hello, world!" program and use it to introduce the structure of a Java program: classes, the main method, and statements. We learn how to produce output using System.out.println, including how to handle text (string literals) and special characters (escape sequences). We also distinguish between print and println. Additionally, we touch on rules for naming things (identifiers and Java keywords) and the importance of writing readable code using comments and proper formatting. The section concludes with a more complex example program (DrawFigures1) that prints a multi-line text figure, demonstrating how Java statements can be combined to produce intricate output, and highlighting some redundancy that we will later address.

Key Concepts and Learning Objectives
  • Understand the basic structure of a Java program: everything is inside a class, and execution starts from the main method.
  • Learn how to write output to the console using System.out.println and System.out.print.
  • Know what a string literal is (text in quotes) and how to include special characters in output using escape sequences (like \" for quotes, \\ for backslash, \n for new line).
  • Distinguish between print vs println and know when to use each (continuing output on the same line vs moving to a new line).
  • Recognize identifiers (names for classes, methods, etc.) and understand the basic naming rules and conventions in Java, as well as which words are reserved keywords that you cannot use as names.
  • Appreciate the role of comments in code for documentation, and understand basic comment syntax (// for single-line, /* ... */ for multi-line).
  • Embrace the importance of readability and style: proper indentation, line breaks, and naming make code easier to understand (both for yourself and others).
  • Be able to read and write a simple Java program that prints multiple lines of output, and be introduced to an example (DrawFigures1) that prints a complex ASCII art figure.
New Terms (Definitions)
Class
A fundamental Java structure that contains code in the form of methods (and sometimes fields). Every Java program is defined inside a class.
Method
A named sequence of statements (a small sub-program). Methods let you reuse behaviour and organise code.
main Method
The entry point of a Java application. Its header is always ; the JVM looks for it when a program starts.
Statement
A single command or instruction in a program. In Java, statements end with a semicolon--for example .
String Literal
Text enclosed in double quotes, such as .
Escape Sequence
A special sequence inside a string that represents a character that is hard to type directly, such as for a quote or for a new line.
System.out
The standard output stream provided by Java. Calling its or methods writes text to the console.
println vs print
Both methods output text. adds a newline at the end of the output whereas leaves the cursor on the same line.
Identifier
A name you assign in code (for a class, method, or variable). Identifiers are case-sensitive and cannot be Java keywords or contain spaces.
Keyword
A word that has a reserved meaning in Java, such as , , or , and therefore cannot be used as an identifier.
Comment
Text ignored by the compiler that documents code. Java supports for single-line and for block comments.
Whitespace/Indentation
Spaces, tabs, and blank lines that organise code for readability. Java ignores most extra whitespace, but humans rely on it to understand structure.

Self-Check Questions (Toggle to reveal answers)

  • Q: Why does every Java application need amainmethod?

    Show Answer

    The main method is the entry point of a Java program. When you run a class, the Java Virtual Machine looks for the public static void main(String[] args) method to know where to begin execution. Without a main method, the JVM wouldn't know what to execute first, so the program wouldn't run (unless it's a special context like Android or applets which have different entry points).

  • Q: What is the output of the following code?
    java System.out.print("Hello"); System.out.print("World"); System.out.println("!"); System.out.println("Hello again!");

    Show Answer

    The output will be:
    HelloWorld!
    Hello again!
    Because System.out.print("Hello") and System.out.print("World") print on the same line (no newline in between), the text will concatenate as HelloWorld. The following println("!") will print ! immediately after "World" and then move to a new line. Finally, println("Hello again!") prints Hello again! on a new line.

Tutorial and Code-Along

Now that we've set up the basics of how Java programs run, let's zoom in on writing Java code itself. We'll start by examining the simplest useful program - the classic "Hello, world!" - and then build up to more interesting output.

1. Our First Java Program Explained - Hello, world!


It's a tradition in computer science to begin with a program that prints "Hello, world!" on the screen. You actually ran a variant of this in Section 1.1. Let's look at a canonical example and break it down:

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}

If you create a file Hello.java with the above code and run it, it will output:

Hello, world!

Let's dissect the code:

  • Class definition (public class Hello { ... }): This defines a class named Hello. The keyword public means this class is accessible to any other code (we won't worry about the details of public vs others yet). The curly braces { } enclose the contents of the class. Java uses { and } as grouping symbols - every opening brace must have a corresponding closing brace. In VS Code, notice how it highlights matching braces which helps keep them paired.

  • Main method definition (public static void main(String[] args) { ... }): Inside the class, we have this exact method header. For now, treat public static void main(String[] args) as boilerplate that you need to include so your program can run. A quick idea of each part:

  • public: means the method is publicly accessible (just like the class).

  • static: means this method belongs to the class itself rather than an instance of the class (we'll explore this later; main must be static so it can run without an object).

  • void: means the method does not return any value.

  • main: the name of the method. This must be exactly main for the program entry point.

  • (String[] args): indicates that the main method takes an array of String as parameters. These would be any command-line arguments passed to the program. We won't use args much initially, but it needs to be there to match the expected signature.

  • The method's body is enclosed in its own { } inside the class.

  • Statement inside main (System.out.println("Hello, world!");): This is a single Java statement. System.out.println is a command that prints a line of text to the console. The text to print is given in parentheses as a string literal: "Hello, world!". Don't forget the semicolon ; at the end - in Java, semicolons mark the end of each statement (think of it like a period in a sentence, but for code).

When the program runs, it executes the main method: encountering System.out.println("Hello, world!"); causes the message to be displayed, then the program ends (since there are no more statements).

2. Printing Text and Escape Sequences


So System.out.println is our primary tool for now to produce output. You can pass any text you want between the quotes and it will print exactly that text (minus the quotes). There are a couple of nuances:

  • If you want to print multiple pieces of text or values on the same line, you can either use multiple print statements or use the + operator to concatenate (join) strings. For now, we'll stick to one string literal per println for simplicity.
  • If you want a newline (line break) in the middle of a string, you can either use two println calls or use the escape sequence \n inside a string. For example:

  • System.out.println("Hello,\nworld!");

    would output: Hello, world!

    because \n is interpreted as "new line".

  • If you need to print a double quote character ", you cannot just put a raw " inside a string literal, because that would be seen as the end of the string. Instead, use the escape sequence \". For example:

  • System.out.println("He said, "Java is fun."");

    outputs: He said, "Java is fun."

  • Similarly, to print a single backslash \, use \\ in the string literal. For example:

  • System.out.println("C:\Users\Public");

    might output: C:\Users\Public

    (because each \\ in the code becomes a single \ in the output).

Here the quote marks around Java is cool! appear because we used \" in the string.

3. Using Special Characters and Formatting Output


Escape sequences allow you to format output beyond plain characters. A few common escape sequences in Java: - \n - newline - \" - double quote - \\ - backslash - \t - tab character (adds horizontal tab spacing)

Feel free to experiment with these. For instance:

System.out.println("Item\tPrice");
System.out.println("Pen\t$1.20");
System.out.println("Notebook\t$2.50");

This would output a simple tab-separated list:

Item    Price
Pen     $1.20
Notebook        $2.50

(The spacing might align differently depending on text length, but you get the idea: \t jumps to the next tab stop.)

4. Understanding Identifiers and Keywords


When writing Java code, you'll be coming up with names for classes, methods, and later variables. These names are identifiers. There are a few rules and conventions: - Rules: Identifiers can include letters, digits, underscores _, and dollar signs $. However, they cannot start with a digit, and they cannot be a reserved Java keyword. Also, no spaces or other special characters are allowed. For example, myValue and value2 are valid, but 2value is not (starts with digit), my value is not (contains space), and class is not (keyword). - Conventions: By convention, Java class names start with a capital letter (e.g., Hello, MyComputer) and use capital letters to start each new word (CamelCase). Methods and variables start with a lowercase letter and also use camelCase (e.g., println is a method name). Constants (which we'll see later) are usually in ALL_CAPS. Following conventions isn't required by the compiler, but it's the standard style and makes your code more readable to others.

In the Hello program above: - Hello is an identifier (class name). - args is an identifier (the name of the parameter in main). - Words like public, class, static, void, String are not identifiers you chose; they are part of Java's vocabulary (keywords or class names from the Java API). For example, class and public are keywords. You'll get to know many of these keywords as you continue; for now, just remember you can't use them for your own names.

5. Comments and Readability


As programs grow beyond a few lines, readability becomes very important. Code is read by humans (often more times than it's written), so we follow formatting conventions: - Indent inside braces: The contents of a class are indented 4 spaces (or a tab, depending on settings) inside the class braces. Contents of methods are indented inside the method braces, and so on. This visual hierarchy makes it easy to see what's inside what. - One statement per line: Don't cram multiple ; separated statements on one line. Use a new line for each. - Use blank lines to separate logical sections of code (for instance, a block of variable declarations from a block of computations, or to separate methods). - Choose meaningful names for classes and methods that reflect their purpose. - Use comments to explain non-obvious parts of code or the overall purpose of the program.

A comment is a note in the source code for humans. Java ignores comments when running the program. There are two main types: - Single-line comment: start with //. Everything after // on that line is ignored. For example:

// This is a comment explaining the next line of code
System.out.println("Hi"); // This comment is at end of a line

- Multi-line (block) comment: start with /* and end with */. Everything in between is ignored, even if it spans multiple lines. For example:

/* 
   This is a block comment.
   It can span multiple lines.
*/
System.out.println("Hi");

Block comments are often used for longer explanations or disabling large sections of code during testing.

An example of poor vs. good formatting


Consider this poorly formatted (though technically valid) Java program:

public class Ugly {public static void main(String[] args) {System.out.print("How");System.out.print("short");System.out.println("am I?");}}

If you try to read this, it's a mess! The program itself just prints "Howshortam I?" (actually all concatenated because they forgot spaces). It's hard to tell where one statement ends and the next begins, and where the class or method boundaries are. Now look at a well-formatted version:

public class Ugly {
    public static void main(String[] args) {
        System.out.print("How");
        System.out.print(" short");
        System.out.println(" am I?");
    }
}

Now we can clearly see: - The class Ugly and its opening {, with a matching closing } aligned under the p in public. - The main method inside, indented. - Each statement on its own line, indented inside main. - We also fixed the output to include spaces so it prints How short am I? instead of Howshortam I?. (Concatenation of strings in separate print statements does not add spaces or punctuation automatically - you have to include them in the strings.)

Now You Try - Spot and Fix Formatting Issues


Below is a snippet of code. It has correct syntax (it will run), but the style is intentionally bad. Identify at least three style issues and think about how you'd fix them:

public class Greeter {public static void main(String[] args) {System.out.println("Hello, CS1 Students!");System.out.println("Welcome to Java");}}

Issues you might notice: - Everything is on one line, which is very hard to read. - No indentation for the contents of the class or method. - It's not immediately obvious where the class or method start and end.

A nicely formatted version would be:

public class Greeter {
    public static void main(String[] args) {
        System.out.println("Hello, CS1 Students!");
        System.out.println("Welcome to Java");
    }
}

Now the structure is clear at a glance.

Remember, the compiler doesn't care about spaces or newlines (except that they can't be in the middle of tokens), but humans do care. As you write code, always format it consistently. The good news is, IDEs like VS Code can auto-format your code (usually there's a command or it happens on save, depending on your setup).

6. A Complex Example: DrawFigures1


So far, we've focused on very small programs with one or two print statements. Let's push this a bit further and see a program that produces a more elaborate output. This will reinforce everything we've learned and also set the stage for the next sections where we'll learn to structure code better.

Consider the following program DrawFigures1 (imagine we saved it as DrawFigures1.java). It uses a bunch of System.out.println calls to draw a text figure:

public class DrawFigures1 {
    public static void main(String[] args) {
        System.out.println("  /\\");
        System.out.println(" /  \\");
        System.out.println("/    \\");
        System.out.println("\\    /");
        System.out.println(" \\  /");
        System.out.println("  \\/");
        System.out.println();
        System.out.println("\\    /");
        System.out.println(" \\  /");
        System.out.println("  \\/");
        System.out.println("  /\\");
        System.out.println(" /  \\");
        System.out.println("/    \\");
        System.out.println();
        System.out.println("  /\\");
        System.out.println(" /  \\");
        System.out.println("/    \\");
        System.out.println("+------+");
        System.out.println("|      |");
        System.out.println("|      |");
        System.out.println("+------+");
        System.out.println("|United|");
        System.out.println("|States|");
        System.out.println("+------+");
        System.out.println("|      |");
        System.out.println("|      |");
        System.out.println("+------+");
        System.out.println("  /\\");
        System.out.println(" /  \\");
        System.out.println("/    \\");
    }
}

(Take a deep breath - that's a lot of lines!)

Notice how the string literals use \\ whenever we want a single backslash in the output, and how some lines have leading spaces to position the shapes correctly. We also use blank System.out.println(); statements to create blank lines in the output (there are two println(); calls with empty strings or no string to separate figure sections).

This program is correct in that it produces the intended output. However, it's clearly very repetitive and long. We have many lines of code that are almost the same, just with varying amounts of spaces. At this point, you might not know how to avoid that repetition - but you might sense that there should be a better way than copying and pasting those println statements. You're absolutely right! Imagine if the above code were not neatly indented or spaced out - it would be almost impossible to debug. Thanks to indentation, you can match each output line in the code to the shape in the output visually. - Show a real use of escape sequences (notice all those \\ in the code that produce single \ in output). - Emphasize that the Java compiler doesn't mind how many lines of code you write or if you repeat yourself - but as programmers, we mind, because it's error-prone and tedious. We'll soon fix that with better techniques.
