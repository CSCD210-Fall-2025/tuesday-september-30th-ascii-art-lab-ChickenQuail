Quick Reference
Summary

This section introduces what computer programs are and why we write them. It covers the fundamental distinction between a computer's hardware (physical components) and software (programs/instructions), explains how information is represented in the digital realm (using 0s and 1s), and outlines the steps in creating and running a program. It also discusses why Java is the language of choice in this course and what tools are needed to write and run Java programs.

Key Concepts and Learning Objectives
  • Understand why programming is important and how it enables computers to perform useful tasks.
  • Identify the major hardware components of a computer (CPU, memory, storage, etc.) versus software components (applications, the OS, code).
  • Grasp the idea of the digital realm - that all data (text, images, etc.) is represented in binary (0s and 1s).
  • Learn why Java is widely used and suitable for beginners.
  • Know the process of programming: writing source code, compiling it, and running the program.
  • Become familiar with the Java programming environment (the tools like JDK and an IDE such as VS Code needed to write and execute Java code).
New Terms (Definitions)
Computer
An electronic device that stores and processes data according to instructions (programs).
Program (Software)
A set of instructions written in a programming language that a computer can execute to perform a specific task.
Hardware
The physical components of a computer system such as the CPU, memory, storage, keyboard and monitor.
CPU (Central Processing Unit)
The "brain" of the computer that executes instructions and performs calculations.
Memory (RAM)
Short-term, volatile storage where programs and data are loaded for quick access by the CPU.
Storage
Long-term memory (for example hard drives or SSDs) where data and programs are kept persistently.
Digital (Binary)
A system of representing data using two states, 0 and 1. All numbers, text, images and sound in a computer are encoded in binary form.
Source Code
The human-readable instructions written by a programmer, typically in files such as .
Compiler
A tool that translates source code into a lower-level form (for Java, bytecode) and checks for syntax errors.
Bytecode
The intermediate compiled code that Java produces in files and that the JVM can execute on any platform.
Java Virtual Machine (JVM)
A virtual computer that runs Java bytecode, allowing Java programs to be portable across different operating systems.
IDE (Integrated Development Environment)
Software that provides a convenient way to write, edit, compile and run programs--for example Visual Studio Code with Java extensions.

Self-Check Questions (Toggle to reveal answers)

  • Q: What is the difference between hardware and software?

    Show Answer

    Hardware refers to the physical components of a computer (CPU, memory, keyboard, etc.), whereas software refers to the programs and instructions that run on the hardware (applications, operating system, code). Hardware is tangible; software is intangible logic that tells the hardware what to do.

  • Q: Why do computers use binary (0s and 1s) to represent data?

    Show Answer

    Computers use binary because their electronic circuits have two distinct states (on/off, represented by 1/0). Using binary is a reliable way for digital circuits to store and manipulate all kinds of information. Any data (numbers, letters, images, etc.) can be encoded in binary form for the computer to process.

  • Q: What are the basic steps to write and run a Java program?

    Show Answer

    First, you write the program's source code in a .java file using a text editor or IDE. Next, you compile the source code using the Java compiler (javac), which checks for errors and translates the code into Java bytecode (a .class file). If compilation succeeds, you then run the program by launching the Java interpreter (java), which executes the bytecode on the Java Virtual Machine, producing the program's output.

Tutorial and Code-Along
1. Computers, Programs, and Why We Need Programming


Computers are everywhere - in our phones, laptops, cars, and even kitchen appliances. But a computer on its own is just machinery. What actually makes a computer useful is a program: a set of instructions telling the hardware what to do. A computer program can be anything from a simple calculator app to a complex video game. By writing programs, we control the computer's behavior and solve problems or perform tasks automatically. This is why programming is so powerful: it allows us to harness the speed and accuracy of computers to do work for us.

2. Hardware vs. Software


A good way to think about a computer is to compare it to a cooking scenario. The computer's hardware is like the kitchen with all its equipment (oven, fridge, utensils), and the program (software) is like the recipe that tells the cook (CPU) what to do with that equipment. The CPU (Central Processing Unit) is the master chef of the computer that executes instructions one by one. It uses ingredients stored in memory (RAM), which is like the countertop space where you place ingredients you're actively working with. There's also long-term storage (like a pantry or fridge - on a computer this would be a hard drive or SSD) where data is kept for later use. Input devices (keyboard, mouse) and output devices (monitor, printer) are how we interact with the computer, like the ingredients coming in and the finished dish going out.

A computer's hardware is designed to be very flexible: by loading different programs (recipes), the same hardware can perform an incredible range of tasks. One moment your computer can display a game; the next, it can play music or calculate a budget, all by switching programs. Software refers to these programs - the sequences of instructions - that configure the hardware to do a specific job.

3. The Digital Realm - All Data as 0s and 1s


Computers ultimately work in binary. Inside the electronic circuits of a computer, everything is represented by high or low voltage (on or off). This maps naturally to two symbols: 1 (on) and 0 (off). Any piece of information can be broken down into a sequence of bits (binary digits). For example, the letter A is represented by the number 65 in one common scheme (ASCII), which in binary is 1000001. A simple black-and-white image can be represented by 0s and 1s indicating white or black pixels. Even sound is digitized into binary numbers.

Why do we care about binary in a programming class? As a programmer, you mostly work in high-level languages (like Java), not directly with 0s and 1s. However, understanding that everything in a computer is numbers gives you insight into how things work under the hood. For instance, you'll eventually learn how text is encoded in binary or how images are just a bunch of numbers. In Java, you don't have to write binary code - the tools will translate your code into binary for the machine - but this concept is the foundation that makes computing possible.

If you're curious: High-Level vs. Low-Level languages

Early computers had to be programmed in very low-level instructions (essentially writing the binary codes or something called assembly language which closely corresponds to binary). These were error-prone and hard for humans to manage. Modern languages like Java are high-level, which means they are closer to human language and abstract away hardware details. One line of Java might result in many binary instructions executed by the CPU, but we let the compiler and runtime handle that translation. High-level languages improve programmer productivity and portability of code between different hardware. The tradeoff is that you rely on compilers/interpreters to bridge the gap to the binary level.

4. Why Learn Programming (and Why Java)?


Programming is the act of creating these software instructions. Why learn it? Because it empowers you to create solutions, automate tasks, and understand the technology that underpins modern life. Even a little programming knowledge can help you break down problems and think logically. For computer science students, programming is a fundamental skill - it's how we "talk" to computers and make them do useful work.

There are many programming languages out there (Python, C++, JavaScript, etc.), each with its strengths. In this course, we're focusing on Java. Java is a popular language that has been around for decades, used in everything from large enterprise servers to Android phone apps. One reason to use Java is that it's designed to be portable - you can write a Java program on one system and run it on almost any other system with a Java Virtual Machine. It's also a statically-typed language with a strict syntax, which, while a bit verbose, helps catch errors early (at compile time). Java is widely used in industry and has a wealth of libraries and community support. For a beginner, Java's syntax will teach you structured programming and object-oriented principles that apply to many other languages. In short, learning Java will set a solid foundation for your programming journey.

5. The Process of Programming (Write -> Compile -> Run)


Writing a program is a multi-step process. Here's the typical workflow:

  • Write the Code: You start by writing your source code in a text editor or an Integrated Development Environment (IDE). In our class, we use Visual Studio Code (VS Code) with the Java extension, which is an IDE that helps manage and run Java code. You create a file with a .java extension (for example, MyProgram.java). Inside that file, you'll write your instructions using Java's syntax. Don't worry about the details of syntax yet - we'll get to that. At this stage, think of it as writing a recipe for the computer.

  • Compile the Code: Once you have some code written, the next step is to compile it. The compiler (javac in Java) checks your code for any syntax errors - like a grammar checker for programming. If you missed a semicolon or misspelled a command, the compiler will complain and give you an error message. You'll need to fix any such errors until the compiler is happy. If the code has no syntax errors, the compiler translates your Java source into bytecode. Bytecode is a lower-level, platform-independent code that ends up in a file with a .class extension. For example, compiling MyProgram.java produces MyProgram.class. This bytecode isn't exactly human-readable, but it's what the computer (specifically the Java Virtual Machine) will execute.

  • Run the Program: Now it's time to see your program in action. You run the program by invoking the Java interpreter (the JVM). In practice, this might mean clicking a "Run" button in your IDE or typing java MyProgram in a terminal. The JVM will load your compiled .class file and start executing the instructions. If your program produces output (like text on the screen), you'll see it now. If it interacts with a user, this is when it would, for example, wait for input or display a window (later in the course). Running is also where any runtime errors or logical errors will surface (more on those in Section 1.3).

Think of this process like writing and performing a play: you write the script (source code), you have a rehearsal where you fix language errors (compilation), and then you go on stage to perform it (running the program).

7. A Tiny First Program


Let's test that everything is set up correctly by writing a very simple Java program. This program will do nothing fancy - it will just print a message to the screen. We'll use it to verify that we can successfully edit, compile, and run a Java program in VS Code.

Now You Try - Your First Java Program


Open VS Code and create a new file called MyComputer.java. Type the following code into the file:

public class MyComputer {
    public static void main(String[] args) {
        System.out.println("Computers are awesome!");
    }
}

Let's break down what's in this file (we will explore all these parts in more detail in Section 1.2, so don't worry if it's not fully clear yet):

  • public class MyComputer { ... } - This defines a class named MyComputer. In Java, every program lives inside a class. The class name must match your file name (MyComputer.java) exactly, including capitalization.
  • public static void main(String[] args) { ... } - This defines a method named main. The main method is special: it's the entry point of any Java application. When you run the program, the code inside main is what executes first. The weird syntax and words (public static void and the String[] args) can be ignored for now - just treat public static void main(String[] args) as a required "magic header" that you'll use in every program for now.
  • System.out.println("Computers are awesome!"); - This line is inside the main method. It's a command to print the text "Computers are awesome!" to the screen. Every statement in Java ends with a semicolon ;. We'll talk more about System.out.println in the next section; it's the way to produce console output.

Save the file, and then compile/run it. In VS Code, you might simply click the Run button. If everything is set up correctly, the program should compile (with no errors), and you should see the message Computers are awesome! appear in the console/terminal.

Congratulations - you just wrote and ran your first Java program! * It might not seem like much, but you've gone through the same steps professionals use to build software: writing code, compiling it, and running it to see the result.

If this didn't work (for example, if you got an error message), double-check that you copied the code exactly, including capitalization and punctuation. Common mistakes for first programs include: - Misspelling main or System (Java is case-sensitive, so Main or system will not work). - Missing a curly brace { or } or a semicolon ;. - Saving the file with the wrong name (filename must exactly match the class name).
